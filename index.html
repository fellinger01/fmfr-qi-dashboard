Clinical Reference — NHTSA Kupas 2017
Lights & Sirens Use by EMS — Clinical Decision Reference
Kupas DF · NHTSA/OEMS · May 2017 · View Source PDF ↗
FMFR Transport Rate: 2.9%
NHTSA Target: <5%
✓ MAY BE JUSTIFIED
▸ Tenuous/compromised airway
▸ Rapidly decompensating vitals
▸ STEMI — only if cath lab ready to receive
▸ Cardiac arrest with ROSC en route
▸ Stroke — early notification more impactful
✗ EVIDENCE DOES NOT SUPPORT
▸ Stable vitals after EMS interventions
▸ Nursing home / SNF transfers
▸ Most falls, orthopedic injuries
▸ Behavioral / psychiatric — siren may worsen
▸ Obvious death / field termination
🏛 PENNSYLVANIA — 28 Pa.C.S. §1027.3(i)
▸ Full stop required at all red lights and stop signs
▸ Speed limit may not be exceeded even with L&S active
▸ L&S transport restricted to situations where outcome may benefit
▸ Both lights and siren required simultaneously
📥 ESO Data Import — New Review Period
In Windows Explorer, find each ESO CSV export and drag it directly onto the zone below. Importing starts a fresh review period.
Tip: Open Windows Explorer (Win+E), navigate to your Downloads or ESO export folder, drag the file onto the box.
STEP 1 — OPERATIONAL / RESPONSE DATA CSV
ESO: Reports → Operational Data for QI Dashboard — timestamps, crew, LOS, EMD complaint, shift, scene time, delays.
⚠ Add these columns to your report for full L&S detection: Transport Mode Lights & Sirens Use · Response Mode Lights & Sirens Use · Transport Disposition

STEP 2 — CLINICAL / TREATMENT DATA CSV
ESO: Reports → QI Dashboard Clinical Information — impressions, treatments, L&S use, transport disposition, injury, mass casualty.
⚠ Critical for accurate L&S counts: This report must include ALL incidents, not just those with treatment entries. In ESO, edit this report and remove any filter limiting rows to incidents with treatments. If the exported file has far fewer rows than total incidents (e.g. 246 rows for 618 incidents), L&S counts will be severely understated. The import log will warn you if coverage is below 50%.
